# BOSSNYUMBA App Localization (i18n)

This directory holds the localization resources for `bossnyumba_app`.

## Supported locales

| Code | Language | Notes |
|------|----------|-------|
| `en` | English  | Source / template locale |
| `sw` | Swahili  | Tanzania + Kenya lingua franca |

## Files

- `app_en.arb` - English source strings (this is the **template** - every
  new key must be added here first).
- `app_sw.arb` - Swahili translations. Must stay in sync key-for-key with
  `app_en.arb`.
- `generated/app_localizations.dart` - auto-generated by `flutter gen-l10n`
  from the ARB files (do not edit by hand - it will be overwritten).
- `README.md` - this file.

Configuration lives in `apps/bossnyumba_app/l10n.yaml`.

## Adding a new string

1. Open `lib/l10n/app_en.arb` and add a new key plus its `@description`
   metadata block. Use camelCase descriptive keys grouped by feature area,
   e.g. `paymentsPayNow`, `maintenanceStatusCompleted`.
2. If the string has runtime variables, declare them in the
   `placeholders` section of the `@key` metadata, e.g.:

   ```json
   "welcomeUser": "Welcome, {name}",
   "@welcomeUser": {
     "description": "Greeting on the home screen",
     "placeholders": {
       "name": { "type": "String", "example": "Jane" }
     }
   }
   ```

3. Add the same key to `lib/l10n/app_sw.arb` with the Swahili translation.
   Key parity is enforced - missing keys will fail codegen.
4. Regenerate the Dart class:

   ```sh
   cd apps/bossnyumba_app
   flutter gen-l10n
   ```

5. Use the new key in a widget:

   ```dart
   import '../../l10n/generated/app_localizations.dart';

   final l10n = AppLocalizations.of(context);
   Text(l10n.welcomeUser(session.firstName));
   ```

## Wiring (already done in `lib/app.dart`)

```dart
MaterialApp.router(
  localizationsDelegates: const [
    AppLocalizations.delegate,
    GlobalMaterialLocalizations.delegate,
    GlobalWidgetsLocalizations.delegate,
    GlobalCupertinoLocalizations.delegate,
  ],
  supportedLocales: const [Locale('en'), Locale('sw')],
  // ...
)
```

## Translation guidelines for Swahili

- Use standard East African Swahili (KiSwahili sanifu), not Sheng.
- Keep UI labels short - Swahili words tend to be longer than English.
- Preserve placeholders exactly: `{amount}`, `{dueDate}`, etc.
- Preserve punctuation and formatting tokens like `#` in work order numbers.
- Preserve brand names: `BOSSNYUMBA`, `M-Pesa`.
- When unsure of a technical term, leave a TODO comment in the PR and ping
  a native speaker from the Tanzania/Kenya team for review.

## Notification & document templates

Notification templates under `services/notifications/src/templates/` already
ship EN + SW copy. The ARB keys under `notification*` mirror those templates
so the app and the notification service stay in sync. If you change a
template, update the matching ARB keys in the same PR.

## Lint / CI check

The key parity between `app_en.arb` and `app_sw.arb` can be checked with:

```sh
python3 -c "
import json
en = {k for k in json.load(open('lib/l10n/app_en.arb')) if not k.startswith('@')}
sw = {k for k in json.load(open('lib/l10n/app_sw.arb')) if not k.startswith('@')}
missing = en - sw
assert not missing, f'Missing Swahili translations: {missing}'
print(f'OK - {len(en)} keys in both locales')
"
```
